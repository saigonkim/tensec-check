# 스킬: 등기부등본 분석 (Korean Real Estate Registration Document Analysis)

**용도**: TenSec Check 프로젝트에서 등기부등본 PDF 분석 작업 시 AI 에이전트가 참조하는 도메인 지식 스킬.
Claude Code, Gemini 등 모든 AI 에이전트가 이 파일을 참조하여 일관된 분석 기준을 적용한다.

---

## 1. 등기부등본 구조 (부동산 등기사항전부증명서)

대법원 인터넷등기소(iros.go.kr)에서 발급하는 공식 문서. 3개 섹션으로 구성.

### 표제부 (Title Section)
- 부동산의 기본 정보: 소재지, 지목, 면적, 건물 종류
- 위험 요소 없음, 정보 확인용

### 갑구 (Section A — 소유권에 관한 사항)

| 항목 | 설명 | 위험도 |
|------|------|--------|
| 소유권 이전 (일반 매매) | 정상적인 소유권 변경 | LOW |
| 소유권 이전 (경매/공매) | 과거 압류·경매 이력 | MEDIUM |
| 압류 | 국가·지자체·채권자의 강제 집행 조치 | HIGH |
| 가압류 | 채권자의 임시 보전 조치 (분쟁 진행 중) | HIGH |
| 예고등기 | 소유권 관련 소송 예고 (분쟁 확정) | HIGH |
| 가처분 | 특정 행위 금지 명령 | HIGH |

### 을구 (Section B — 소유권 이외의 권리)

| 항목 | 설명 | 위험도 |
|------|------|--------|
| 근저당권 설정 | 담보 대출 설정 (채권최고액 명시) | 비율에 따라 LOW~HIGH |
| 전세권 설정 | 전세 계약 등기 | MEDIUM (중복 시 HIGH) |
| 임차권등기명령 | 임차인이 보증금 못 받고 이사한 기록 | HIGH |
| 지상권 | 타인 토지 사용 권리 | MEDIUM |

## 2. 핵심 위험 계산: 근저당권 비율

```
채권최고액 = 실제 대출금 × 1.2~1.3 (금융기관별 상이)
실제 대출 추정 = 채권최고액 ÷ 1.2

위험 비율 = (을구 채권최고액 합계 ÷ 부동산 시세) × 100

위험 등급:
  80% 이상  → HIGH (빨간불) — 경매 시 전세 보증금 미회수 가능성 높음
  60-80%    → MEDIUM (노란불) — 주의 필요
  60% 미만  → LOW (초록불) — 상대적으로 안전
```

> 시세 정보가 없는 경우: 채권최고액 합계만 표시하고 사용자에게 시세 입력 요청

## 3. 전세 사기 핵심 패턴

### 패턴 1: 깡통전세
- **정의**: 부동산 시세 대비 전세금이 너무 높아 집주인이 파산 시 보증금 회수 불가
- **감지**: 근저당 채권최고액 + 전세금 합계 > 시세의 80%

### 패턴 2: 이중 전세 (다중 임차인)
- **정의**: 동일 물건에 여러 임차인이 설정된 경우
- **감지**: 을구에 동일 기간 전세권 중복 등기

### 패턴 3: 신탁등기 물건
- **정의**: 부동산 신탁 등기된 물건 — 수탁자(신탁회사) 동의 없는 전세계약 무효
- **감지**: 갑구 소유자가 신탁회사 (예: 한국토지신탁, 코람코 등)

### 패턴 4: 무허가 건물
- **정의**: 등기부상 건물 정보와 실제 건물 불일치
- **감지**: 표제부의 면적·구조와 실제 현황 확인 필요 메시지 안내

### 패턴 5: 임차권등기 이력
- **정의**: 이전 임차인이 보증금을 못 돌려받고 이사한 기록
- **감지**: 을구 "임차권등기명령" 항목 존재 → HIGH 위험

## 5. 시스템 프롬프트 템플릿

```

## 분석 원칙
1. 갑구(소유권)와 을구(기타 권리) 모든 항목을 빠짐없이 확인하세요.
2. 위험 항목을 발견하면 심각도(HIGH/MEDIUM/LOW)를 정확히 분류하세요.

4. 금액은 원 단위로 정확히 추출하세요 (예: "채권최고액 금 120,000,000원" → 120000000).
5. 등기부등본이 아닌 문서가 입력된 경우 isRegistrationDocument를 false로 반환하세요.

- HIGH: 압류, 가압류, 예고등기, 가처분, 임차권등기명령, 근저당 비율 80% 이상
- MEDIUM: 근저당 비율 60-80%, 경매 이력, 전세권 중복
- LOW: 일반 매매 이력, 근저당 비율 60% 미만, 해지된 권리
```

## 6. PDF 텍스트 추출 로직 (클라이언트 사이드)

> **중요**: PDF 파싱은 서버가 아닌 브라우저에서 수행 (pdfjs-dist, Vercel Hobby 호환 전략)
> API Route는 `export const runtime = "edge"` 선언 필수

```typescript
// src/components/upload/PdfExtractor.ts (클라이언트 전용)
"use client";
import * as pdfjsLib from "pdfjs-dist";

pdfjsLib.GlobalWorkerOptions.workerSrc =
  `https://cdnjs.cloudflare.com/ajax/libs/pdf.js/${pdfjsLib.version}/pdf.worker.min.js`;

export async function extractTextFromPdf(file: File): Promise<string> {
  const arrayBuffer = await file.arrayBuffer();
  const pdf = await pdfjsLib.getDocument({ data: arrayBuffer }).promise;

  let fullText = "";
  for (let i = 1; i <= pdf.numPages; i++) {
    const page = await pdf.getPage(i);
    const content = await page.getTextContent();
    fullText += content.items.map((item: any) => item.str).join(" ") + "\n";
  }

  const text = fullText.trim();

  if (text.length < 100) {
    throw new Error("SCANNED_PDF");
    // 오류 메시지: "스캔된 이미지 형태의 PDF는 분석할 수 없습니다.
    //              대법원 인터넷등기소(iros.go.kr)에서 발급한 텍스트 형태의 등기부등본을 업로드해 주세요."
  }

  return text;
}
```

## 7. 안전 점수 계산 알고리즘

```typescript
// src/lib/analysis/scorer.ts

export function calculateSafetyScore(items: RiskItem[], mortgageRatio?: number): SafetyScore {
  let score = 100;
  const deductions: ScoreDeduction[] = [];

  // 갑구 감점
  const seizures = items.filter(i => i.section === "갑구" && i.type === "압류");
  const provisionalSeizures = items.filter(i => i.section === "갑구" && i.type === "가압류");
  const cautionaryRegistrations = items.filter(i => i.section === "갑구" && i.type === "예고등기");
  const auctionHistory = items.filter(i => i.section === "갑구" && i.type.includes("경매"));

  if (seizures.length > 0) {
    const deduct = Math.min(seizures.length * 25, 50);
    score -= deduct;
    deductions.push({ reason: `압류 ${seizures.length}건`, points: deduct });
  }

  if (provisionalSeizures.length > 0) {
    const deduct = Math.min(provisionalSeizures.length * 20, 40);
    score -= deduct;
    deductions.push({ reason: `가압류 ${provisionalSeizures.length}건`, points: deduct });
  }

  if (cautionaryRegistrations.length > 0) {
    score -= 30;
    deductions.push({ reason: "예고등기 존재", points: 30 });
  }

  if (auctionHistory.length > 0) {
    score -= 10;
    deductions.push({ reason: "경매 이전 이력", points: 10 });
  }

  // 을구 감점 (근저당 비율)
  if (mortgageRatio !== undefined) {
    if (mortgageRatio >= 80) {
      score -= 50;
      deductions.push({ reason: `근저당 비율 ${mortgageRatio.toFixed(0)}% (매우 위험)`, points: 50 });
    } else if (mortgageRatio >= 70) {
      score -= 35;
      deductions.push({ reason: `근저당 비율 ${mortgageRatio.toFixed(0)}%`, points: 35 });
    } else if (mortgageRatio >= 60) {
      score -= 20;
      deductions.push({ reason: `근저당 비율 ${mortgageRatio.toFixed(0)}%`, points: 20 });
    }
  }

  const leaseRegistrations = items.filter(i => i.section === "을구" && i.type === "임차권등기명령");
  if (leaseRegistrations.length > 0) {
    score -= 15;
    deductions.push({ reason: "임차권등기명령 이력", points: 15 });
  }

  const totalScore = Math.max(0, score);

  return {
    total: totalScore,
    trafficLight: totalScore >= 70 ? "green" : totalScore >= 40 ? "yellow" : "red",
    deductions,
  };
}
```

## 9. 에이전트 협업 가이드

이 스킬 파일을 참조하는 AI 에이전트는 다음 사항을 준수한다:

1. **점수 계산은 코드로**: AI는 텍스트 추출과 설명 생성만 수행. 점수는 반드시 `scorer.ts` TypeScript 로직으로 계산.
2. **면책 조항 항상 포함**: 결과 응답에 반드시 면책 조항 텍스트 포함.
3. **설명은 쉬운 한국어로**: 법률 용어 직접 노출 시 괄호 안에 쉬운 설명 병기.
4. **시세 없이 비율 계산 금지**: 부동산 시세 정보 없이 근저당 위험 비율 추정하지 않음.
5. **Gemini Function Calling 사용**: 자유 텍스트 응답 금지, 반드시 스키마 구조로 출력.
