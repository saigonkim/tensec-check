## 1. 프로젝트 개요 (Project Overview)

### 서비스 정보
- **서비스명**: TenSec Check (텐섹 체크)
- **슬로건**: "앱 설치도, 광고도 없습니다. 업로드 한 번으로 10초 만에 부동산 등기부등본을 확인하세요."
- **전략**: No App, Just Link — 설치·가입·광고 없는 극강의 미니멀리즘 서비스

### 탄생 배경
2023-2025년 한국에서 전세 사기 피해가 급증하며 사회적 문제로 대두되었습니다. 피해자 대부분은 등기부등본을 발급받았음에도 그 내용을 해석하지 못해 위험을 인지하지 못했습니다.

**핵심 Pain Points**:
1. "이 집, 전세 사기 위험 없을까?" — 일반인은 등기부등본을 봐도 무엇이 위험한지 모름
2. "근저당이 많으면 얼마나 위험한 거야?" — 금융 개념과 법률 용어의 이중 장벽
3. "전문가에게 물어보려니 비용이 부담됨" — 계약 전 전문가 자문 접근성 낮음

### 서비스 정의
본 서비스는 **정보 제공만을 목적**으로 하며, 법적 효력을 가진 권리분석이 아닙니다. AI가 내린 분석 결과는 참고용이며, 최종 계약 결정의 책임은 사용자에게 있습니다.

## 2. 타겟 사용자 (Target Users)

### Primary User
- **대상**: 전세/매매 계약을 앞둔 20-40대 임차인 및 매수인
- **특징**: 등기부등본을 발급할 줄은 알지만 해석은 어려움
- **행동 패턴**: 계약 당일 또는 전날 서류 확인, 시간적 여유 없음

### Secondary User
- **대상**: 부모 대신 자녀가 서류를 확인해주려는 가족
- **특징**: 자녀가 IT에 익숙하고 부모 계약을 도와주는 상황

### Anti-Persona (서비스 대상 아님)
- 공인중개사, 법무사, 변호사 — 이미 스스로 해석 가능
- 투자 목적 전문 투자자 — 더 정밀한 분석 필요

## 3. 사용자 스토리보드 (User Journey)

```
Step 1: 유입
  검색("전세 사기 확인 방법"), SNS 공유, 지인 추천으로 사이트 접속

Step 2: 가치 인지
  히어로 섹션: 슬로건 + "PDF 업로드 → 10초 분석" 시각적 전달
  신뢰 지표: "저장 안 함 | 광고 없음 | 가입 불필요" 배지 표시

Step 3: 면책 동의
  분석 시작 전 ConsentModal 표시
  면책 조항 전문 확인 + "동의하고 분석 시작" 버튼
  동의 없이는 업로드 버튼 비활성화

Step 4: PDF 업로드
  드래그앤드롭 또는 버튼 클릭으로 등기부등본 PDF 선택
  클라이언트 사이드 검증 (PDF 형식, 10MB 제한)
  업로드 완료 즉시 분석 시작

Step 5: 분석 중
  로딩 UI: "파일 읽는 중 → AI 분석 중 → 결과 생성 중" 단계 표시
  예상 소요 시간 안내 (약 10-30초)

Step 6: 결과 확인
  상단: 신호등(빨간/노란/초록) + 안전 점수(0-100)
  중단: 위험 항목 카드 (항목명 + 쉬운 설명 + 심각도)
  하단: AI 종합 의견 + "꼭 확인하세요" 체크리스트

Step 7: 전문가 연결
  빨간불/노란불: "전문가에게 확인받기" CTA 강조 표시
  초록불: "그래도 계약 전 전문가 확인 권장" 보조 CTA
```

## 4. 핵심 기능 명세 (Feature Specifications)

### 4.1 PDF 업로드 시스템

| 항목 | 명세 |
|------|------|
| 허용 파일 형식 | PDF만 허용 (MIME: application/pdf) |
| 최대 파일 크기 | 10MB |
| 업로드 방식 | 드래그앤드롭 + 버튼 클릭 모두 지원 |
| 서버 저장 여부 | **저장 안 함** — 메모리에서 처리 후 즉시 폐기 |
| 클라이언트 검증 | MIME 타입, 파일 크기 즉시 피드백 |
| 에러 처리 | 스캔본 PDF, 암호화 PDF, 텍스트 없는 PDF 각각 안내 |

**스캔본 PDF 처리**: 텍스트 추출 결과가 100자 미만이면 오류 안내
→ "스캔된 이미지 형태의 PDF는 분석할 수 없습니다. 대법원 인터넷등기소(iros.go.kr)에서 발급한 텍스트 형태의 등기부등본을 업로드해 주세요."

### 4.2 AI 분석 엔진

**사용 AI**: Google Gemini (gemini-1.5-pro 또는 gemini-2.0-flash)

**추출 대상 항목**:

| 구분 | 항목 | 설명 |
|------|------|------|
| 갑구 | 압류 | 국가/지자체 체납에 의한 강제조치 |
| 갑구 | 가압류 | 채권자의 임시 보전조치 (분쟁 진행 중) |
| 갑구 | 예고등기 | 소유권 분쟁 예고 (매우 위험) |
| 갑구 | 소유권 이전 방식 | 일반 매매 vs 경매/공매 이력 |
| 을구 | 근저당권 | 채권최고액, 채권자(은행명), 설정일 |
| 을구 | 전세권 | 전세금액, 기간 |
| 을구 | 임차권등기명령 | 보증금 미반환 이력 (위험 신호) |

**출력 형식**: Gemini Function Calling으로 구조화된 JSON (RiskReport 타입)

### 4.3 안전 점수 시스템

점수 계산은 AI가 아닌 TypeScript 로직(`scorer.ts`)으로 수행하여 일관성 보장.

**기본 점수**: 100점

**갑구 감점**:
- 압류 존재: -25점/건 (최대 -50점)
- 가압류 존재: -20점/건 (최대 -40점)
- 예고등기 존재: -30점 (즉시 빨간불 처리)
- 경매/공매 소유권 이전 이력: -10점

**을구 감점**:
- 근저당권 비율 60-70%: -20점
- 근저당권 비율 70-80%: -35점
- 근저당권 비율 80% 이상: -50점 (즉시 빨간불)
- 임차권등기명령 이력: -15점
- 전세권 중복 설정: -10점

**신호등 매핑**:
| 점수 | 신호등 | 의미 |
|------|--------|------|
| 0-39 | 빨간불 🔴 | 높은 위험 — 계약 재검토 강력 권고 |
| 40-69 | 노란불 🟡 | 중간 위험 — 전문가 확인 필요 |
| 70-100 | 초록불 🟢 | 낮은 위험 — 상대적으로 안전 |

> 주의: 근저당권 비율을 계산하려면 사용자가 부동산 시세를 입력하거나 AI가 추정해야 함. MVP에서는 채권최고액 합계만 표시하고 시세 입력 필드를 제공.

### 4.4 결과 표시 UI

**결과 페이지 구성**:
1. **상단 히어로**: 신호등 + 점수 + 한 줄 요약
2. **위험 항목 카드**: 항목명 / 심각도 배지(HIGH·MEDIUM·LOW) / 쉬운 한국어 설명
3. **종합 의견**: AI가 생성한 1-2문장 요약 (전문 용어 없이)
4. **체크리스트**: "계약 전 꼭 확인하세요" 항목 (위험도에 따라 맞춤 생성)
5. **다시 분석하기**: 새 PDF 업로드 버튼

### 4.5 전문가 연결 CTA

| 신호등 | CTA 표시 방식 |
|--------|--------------|
| 빨간불 | 강조 박스 + "지금 바로 전문가와 상담하세요" |
| 노란불 | 결과 하단 + "더 정확한 확인을 원하시면" |
| 초록불 | 보조 텍스트 + "그래도 계약 전 전문가 확인을 권장합니다" |

**연결 방식** (MVP): 이메일 또는 카카오톡 채널 (UTM 파라미터 부착)
**수익화 모델**: 제휴 공인중개사·변호사에게 클릭당 수수료

## 6. 법적 요구사항 (Legal Requirements)

### 6.1 면책 조항 전문 (사용자 동의 필수)

> 본 서비스는 인공지능을 활용한 정보 제공 서비스로, 법적 효력을 가진 권리분석이나 법률 자문이 아닙니다.
>
> 분석 결과는 참고용으로만 활용하시고, 실제 계약 체결 전 반드시 공인중개사 또는 법률 전문가의 확인을 받으시기 바랍니다.
>
> 본 서비스는 분석 결과의 정확성을 보장하지 않으며, AI 분석 결과로 인한 직접적·간접적 손해에 대해 책임을 지지 않습니다.
>
> 최종 계약 결정의 책임은 전적으로 사용자에게 있습니다.

### 6.2 사용자 동의 플로우
- 분석 시작 전 ConsentModal 표시 (건너뛸 수 없음)
- 면책 조항 전문 스크롤 가능 영역에 표시
- "위 내용을 확인하였으며 동의합니다" 체크박스 필수
- 동의 완료 후만 업로드 기능 활성화

### 6.3 개인정보처리방침 (간이)
- 업로드된 PDF는 분석 후 즉시 삭제 (서버 미저장)
- 분석 결과는 세션 내에서만 유지
- 수집하는 개인정보 없음 (로그인 불필요)
- `/privacy` 페이지로 제공

## 7. 디자인 시스템 (Design System)

### 7.1 디자인 방향
- **벤치마크**: 직방, 다방, KB부동산의 신뢰감 있는 UI 패턴
- **원칙**: 깔끔함(Clean), 신뢰감(Trustworthy), 즉시성(Immediate)
- **레이아웃**: 모바일 퍼스트, max-width 640px 기준

### 7.2 색상 팔레트
```css
/* 메인 컬러 — 신뢰감 */
--color-primary:       #1B3A6B;  /* 딥 네이비 */
--color-primary-light: #2E5BA8;  /* 미디엄 블루 */

/* 배경 */
--color-background:    #F8F9FB;  /* 오프 화이트 */
--color-surface:       #FFFFFF;

/* 텍스트 */
--color-text-primary:  #1A1A2E;
--color-text-secondary:#64748B;

/* 신호등 */
--color-red:           #EF4444;
--color-red-bg:        #FEF2F2;
--color-yellow:        #F59E0B;
--color-yellow-bg:     #FFFBEB;
--color-green:         #22C55E;
--color-green-bg:      #F0FDF4;
```

### 7.3 타이포그래피
- **폰트**: Pretendard (한국어 최적화, 직방/다방 사용)
- **로드 방식**: CDN `@import url('https://cdn.jsdelivr.net/gh/orioncactus/pretendard/...')`
- **스케일**: 14px 본문 / 18px 소제목 / 24px 제목 / 32px 히어로

### 7.4 컴포넌트 원칙
- 카드 기반 레이아웃 (rounded-2xl, 부드러운 그림자)
- 충분한 여백 (신뢰감 형성)
- 아이콘: Lucide React
- 애니메이션: CSS transitions만 사용

## 9. 범위 외 (Out of Scope — MVP)

- 회원가입 / 로그인
- 분석 결과 저장 / 히스토리
- 여러 문서 동시 분석
- OCR (스캔본 이미지 PDF 분석)
- 건축물대장, 토지대장 분석 (등기부등본만)
- 특약사항 독소 조항 분석 (Phase 4 이후)
